# Gradio Demo Guide

This guide provides instructions on how to run the Gradio demo for JarvisArt.

## Step 1: Installation
1. **Conda Environment:** Set up the conda environment with required dependencies before running the demo.
```bash
conda create -n jarvisart_demo python=3.11
conda activate jarvisart_demo
pip install -r envs/requirements_demo.txt
git submodule update --init 
cd src/sft
pip install -e .
cd ../../dependence/gradio_image_annotator
pip install -e .
cd ../..
```
2. **Install Adobe Lightroom:** Please download and install Adobe Lightroom on your local machine from the [official website](https://www.adobe.com/products/photoshop-lightroom.html). After installation, sign in using your Adobe account credentials.

> **Note:** Adobe Lightroom is a commercial product and may require a subscription or trial account.

## Step 2: Download Model Weights

To run the Gradio demo, you need to download the model weights from Hugging Face and place them in the correct location:

1. Download the JarvisArt weights from [Hugging Face repository](https://huggingface.co/JarvisArt/JarvisArt-1208)
2. Create the weights directory (if it doesn't exist):
   ```bash
   cd JarvisArt/
   mkdir -p ./checkpoints/pretrained/JarvisArt/
   ```
3. Place the downloaded weight files in the `./checkpoints/pretrained/JarvisArt/` directory
4. If you've placed the model weights in a different location, remember to update the `model_name_or_path` parameter in `src/inference/config/qwen2_vl.yaml` to point to your custom model directory.


## Step 3: Running the Gradio Demo

Once the environment is set up and activated, you can run the Gradio demo with the following command from the root directory of the project:

```bash
cd JarvisArt/
bash demo.sh
```
This will launch a web interface at `http://127.0.0.1:7880`.
> **Notes:**
> - Make sure the model path points to a directory containing a valid JarvisArt model with all necessary files.
> - The web interface will be accessible on port 7880 by default.
> - If you want to customize the model path, port, or other parameters, you can configure them in the `demo.sh` script.

## Step 4: Process the Image

JarvisArt provides two distinct approaches for applying generated editing instructions to your images:

### 4.1 End-to-End Approach: Agent-to-Lightroom Protocol (Recommended)

This approach provides a seamless, fully automated workflow that directly applies the model's editing instructions to your images in Adobe Lightroom with a single click.

**Setup Instructions:**

Before using the one-click send to Lightroom feature, please install and start the Lightroom server by following the [Agent-to-Lightroom Protocol Guideline](./lrc_scripts/clients/agent_to_lightroom/README.md).

Once the Lightroom server is running, configure its URL in the demo UI under "Lightroom Server", or by command-line flags.

**Configuration Options:**

- **Same Machine Setup**
  - Set the Lightroom server URL directly, for example: `http://127.0.0.1:8888` (or your actual port/path).

- **Different Machines Setup**
  1) **Direct Network Access**
     - If the Lightroom server is reachable directly over the network (e.g., has a routable IP or is accessible within the same LAN)
     - Set the Lightroom URL to the server's address and port, for example: `http://<LIGHTROOM_SERVER_IP>:8888`.

  2) **SSH Access Only**
     - Use SSH reverse port forwarding from the Lightroom host (client) to the server running the model (server). Example:
       - With debug logs (for first-time verification):
         ```bash
         ssh -vvv -o ExitOnForwardFailure=yes -N -R 8899:127.0.0.1:8888 user@SERVER_IP
         ```
       - Once verified, you can remove `-vvv` for a quieter run:
         ```bash
         ssh -o ExitOnForwardFailure=yes -N -R 8899:127.0.0.1:8888 user@SERVER_IP
         ```
     - Then, on the model server, set the Lightroom URL in the demo to: `http://127.0.0.1:8899`

**Important Notes:**
- If your Lightroom HTTP endpoint is not at the root path, include the path in the URL (e.g., `http://127.0.0.1:8888/apply`).
- The demo supports remote upload mode. In the UI, keep "Force remote mode (upload files)" enabled (default). Via CLI, use `--lr_remote true`.
- If you use a proxy on the model server, ensure local addresses bypass it (e.g., `NO_PROXY=127.0.0.1,localhost`).

### 4.2 Semi-Automatic Approach: Manual Preset Import

This approach generates Lightroom preset files that you can manually import and apply to your images, providing more control over the editing process.

**Step 1: Convert Lua to LrTemplate Files**

Use the `tools/lua2lrt.py` script to convert Lua files (generated by the model) into LrTemplate files:

```bash
python tools/lua2lrt.py /path/to/your/xxx.lua 
```

> **Note:** 
> - Ensure that your Python environment is properly set up before proceeding.
> - If there are multiple result folders, always use the one with the highest number for the latest output.

**Step 2: Import Generated Presets into Adobe Lightroom**

To use the generated `.lrtemplate` files in Adobe Lightroom:

1. Open Adobe Lightroom.
2. Navigate to the **Presets** panel.
3. Click on the **+** icon.
4. Select **Import Presets**.
5. Choose the `.lrtemplate` file(s) you generated and click **Import**.

The imported presets will now be available in your Presets panel for manual application to your photos.

**Approach Comparison:**
- **End-to-End (4.1)**: Fully automated, one-click application, requires server setup
- **Semi-Automatic (4.2)**: Manual control, no server required, traditional preset workflow
